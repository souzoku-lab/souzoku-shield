## 撮影に使う相談

最初は「曖昧な相談を試す」ボタンで次を入れる。

```text
父が亡くなり、次男が実家を引き継ぐ話になっています。必要な確認を進めてください。
```

追加回答:

```text
次男は父とは別に賃貸住宅で暮らし、本人も配偶者も住宅を所有していません。
```

合格テイクは次の順序が一画面で読めるものだけを使う。

```text
request_clarification
→ 状態変更なしで停止
→ 追加回答
→ select_taker_branch / house_lost
→ fallback なし
→ Evidence / Draft / Review
```

## 審査員に伝える順序

1. 税務制度ではなく「専門判断の再現性」という課題を提示する。
2. AIが情報不足を判断して止まる実画面で、固定フォームとの差を証明する。
3. 決定的コアとCIで、専門判断を検証可能にしたことを示す。
4. HITLとWordで、責任と実務成果へ着地させる。
5. 5,600万円は冒頭の煽りではなく、品質保証が守る具体的影響として使う。

## 表現ルール

- 「専門家でも解決できない」と言わない。`同じ品質で再現し続けるのは困難`と表現する。
- 「Geminiが税務判断する」「すべて自律生成する」と言わない。
- `一つのFunction Callingだけ`と弱点をナレーションで強調しない。
- `5,600万円`は税額・損失ではなく、**課税価格への影響**と必ず表示する。
- `AIが学習・自己改善する`、`あらゆる相続に対応`とは言わない。
- テスト件数は撮影時の最終CI表示と一致させる。追加後に件数が変われば台本も直す。
- UI全景を長く映さず、各カットで証拠を一つだけ拡大する。
- CIカットの`72 tests / secret scan / container smoke`は小さな証拠ラベルにし、件数を主役にしない。
