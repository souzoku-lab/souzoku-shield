# 60秒動画台本

## ナレーション

自宅の取得者によって、課税価格に5,600万円の差が出ます。相続の盾は、税理士の確認と書面作成を支援します。Geminiは一つのFunction Callingで取得者区分だけを選択。決定的コアが、不足資料、リスク、書面添付ドラフトを組み立て、取得者別の確認分岐も提示します。税務判断はAIに任せません。53件のテストをCIで回し、Cloud Runで公開。レビュー待ちで停止し、人の承認後だけWordを出力。曖昧さはGemini、正しさは決定的処理、責任は人へ。

## 画面構成

1. タイトルと個人情報警告を映す。
2. 相続人カードと既定相談文を映す。
3. 「AIエージェントを実行」を押し、Gemini実行トレースを映す。
4. `select_taker_branch`、`house_lost`、fallbackなし、latencyを映す。
5. 課税価格`+56,000,000円`の適用不可アラートを映す。
6. 不足資料、反実仮想分岐、書面添付ドラフトを映す。
7. 総合所見欄が税理士手入力であることを映す。
8. レビュー完了（承認）後だけWord出力できることを映す。
9. GitHub Actions CIの緑とCloud Run公開URLを映す。

## 表現ルール

- `5,600万円`は税額ではなく、課税価格への影響として説明する。
- Geminiは取得者区分だけを選ぶ、と説明する。
- 不足資料、リスク、ドラフトは決定的コアが組み立てる、と説明する。
- 税務判断、総合所見、署名、提出は税理士が担う、と説明する。
- OCR、登記名義自動照合、永続DB、本人認証、永続監査ログは未実装として扱う。
